Task 23: Force Override Evidence
==================================

Force override mechanism:
1. MutationGate: opts.force=true → bypasses deny, returns allowed=true, calls onForceOverride
2. ToolFabric: opts.force=true → bypasses deny, returns warn action, calls onForceOverride

Both gates record ForceOverrideRecord:
  { gate: string, tool: string, reason: string, timestamp: string, override: "force" }

Test coverage:
- "force override bypasses enforced gate" (mutation-gate.test.ts)
  → Bash + rm -rf / + force=true → allowed=true + audit record with gate=MutationGate
- "force override on critical file bypasses and logs audit" (mutation-gate.test.ts)
  → Write + .env + force=true → allowed=true + audit record with gate=MutationGate
- "force override bypasses enforced gate and logs audit trail" (tool-fabric.test.ts)
  → Write in restricted lane + force=true → warn action + audit record with gate=ToolFabric

All 3 force-override tests pass.
Audit trail callback (onForceOverride) is invoked with correct gate/tool/reason.
