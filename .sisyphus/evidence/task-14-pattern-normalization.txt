## Objective

## Changes Made

### 1. Enhanced Normalization Logic (hooks/circuit-breaker.py:42-67)
Added support for:
- `python3 -m X` Ôćĺ `X` (e.g., `python3 -m pytest` Ôćĺ `pytest`)

- `bunx X` Ôćĺ `X` (e.g., `bunx jest` Ôćĺ `jest`)

### 2. Improved Success Clearing Logic (hooks/circuit-breaker.py:170-211)
Implemented `_normalize_tracker_key()` function that:
- Normalizes all tracker keys using the same rules as pattern_key generation
- On success, clears ALL similar pattern variants (not just exact match)
- When clearing `npm test`, also clears `npm run test`, `npx test`, etc.
- When clearing `pytest`, also clears `python3 -m pytest`, `python -m pytest`, etc.

### 3. Added Comprehensive Tests (tests/test_circuit_breaker.py:316-478)

- `test_bunx_jest_normalized_to_jest`: Verifies `bunx jest` normalizes to `jest`
- `test_success_clears_similar_variants`: Verifies success on `npm test` clears `npm run test`

## Key Implementation Details

### Pattern Normalization Flow
1. Command input: `"python3 -m pytest tests/"`
2. Strip prefixes: `"pytest tests/"`
3. Split and filter: `["pytest", "tests/"]`
4. Pattern key: `"Bash:pytest tests/"`

### Success Clearing Flow
1. On success with pattern_key `"Bash:pytest tests/"`
2. Normalize all tracker keys using same rules
3. Find all keys that normalize to `"Bash:pytest tests/"`
4. Remove all matching keys (including `"Bash:python3 -m pytest tests/"`, `"Bash:pytest"`, etc.)
