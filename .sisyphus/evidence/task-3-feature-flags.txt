## Implementation Details

### Resolution Order
1. Environment variable: OMG_{FLAG_NAME.upper()}_ENABLED
   - "0", "false", "no" → False
   - "1", "true", "yes" → True
2. settings.json: _omg.features.{flag_name}
3. default parameter (default=True)

### Caching Strategy
- Module-level dict _FEATURE_CACHE populated on first call
- Prevents repeated file I/O during hook execution
- Cache cleared between test runs for isolation

### Error Handling
- Missing settings.json: returns default
- Malformed JSON: returns default
- Missing _omg section: returns default
- Missing features section: returns default
- No exceptions raised
