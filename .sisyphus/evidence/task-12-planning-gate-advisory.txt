The check_planning_gate() function now:
1. Checks if pending items exist
2. If pending > 0:
   a. Reads .oal/state/.context-pressure.json
   b. Extracts is_high boolean (default: false)
   c. If is_high: returns ([], [advisory_message])
   d. If not high: returns ([block_message], [])

The advisory message format:
"[OAL advisory] Planning gate: {done}/{total} complete, {pending} pending. (demoted: context pressure high)"

This allows the planning gate to be advisory-only when context is exhausted, preventing false blocks
when the user is running out of tokens but still has pending checklist items.
