# Getting Started for Execs

OMG positions AI delivery like an operating system for work: governed, measurable, and easier for teams to trust.

## Why it matters

- Faster path from request to working output
- Safer operations through approval-aware governance
- Clearer accountability through proof and audit surfaces

## Team onboarding flow

1. Validate the environment with `npx omg env doctor`
2. Preview rollout with `npx omg install --plan`
3. Apply when the team is ready with `npx omg install --apply`

## KPIs to watch

- Time-to-first-output
- Successful proof-backed completions
- Reduced risky mutations and rollback events
- Multi-host adoption across Claude, Codex, Gemini, and Kimi

## Leadership takeaway

OMG gives teams an AI Work OS posture: one layer for governed automation, proof-backed execution, and clearer delivery visibility.
