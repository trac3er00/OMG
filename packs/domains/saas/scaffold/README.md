# SaaS Project

Generated by OMG Domain Pack: **saas**

## Quick Start

```bash
# Install dependencies
npm install

# Set up environment
cp .env.example .env
# Edit .env with your actual values

# Start development services
docker compose up -d

# Run database migrations
npm run db:migrate

# Start the development server
npm run dev
```

## Architecture

```
src/
  api/          # API routes (versioned: v1/, v2/)
  auth/         # Authentication and session management
  billing/      # Subscription, payment, and usage metering
  tenants/      # Tenant provisioning, lifecycle, and isolation
  features/     # Feature flag evaluation
  audit/        # Audit logging
  middleware/   # Auth, rate-limiting, tenant-context middleware
  jobs/         # Background job processors (with tenant context)
  lib/          # Shared utilities and clients
```

## Domain Rules

This project is governed by the following OMG SaaS domain rules:

- **Auth & Sessions**: JWT/session management, MFA, service auth
- **Multi-Tenancy**: Isolation models, query safety, lifecycle
- **Billing**: Plans, payments, usage metering, subscription state machine
- **API Design**: Versioning, pagination, idempotency, webhooks
- **Data Isolation**: Storage boundaries, cache isolation, encryption, residency
- **Rate Limiting**: Tiered limits, abuse protection, response headers
- **Feature Flags**: Plan-gated features, lifecycle, safe defaults
- **Audit Logging**: Append-only, tamper-evident, tenant-accessible
- **Security Hardening**: Transport, input validation, secrets, dependencies

## Key Decisions

| Concern          | Default Choice           | Override In            |
| ---------------- | ------------------------ | ---------------------- |
| Tenancy Model    | Row-level isolation      | `TENANCY_MODEL` env    |
| Auth Strategy    | Session + refresh tokens | `AUTH_STRATEGY` env    |
| Payment Provider | Stripe                   | `PAYMENT_PROVIDER` env |
| Feature Flags    | Database-backed          | `FLAG_PROVIDER` env    |
| Rate Limiter     | Redis sliding window     | `RATE_LIMIT_STORE` env |
